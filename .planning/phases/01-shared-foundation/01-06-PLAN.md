`BaseSyncRepository<TModel>` abstract class contract (generic over a feature's model type — Phase 2-6 supply `TModel`):
- Abstract hooks a subclass must implement: `Future<TModel?> getCached(String id)`, `Future<TModel?> getRemote(String id)`, `Future<void> upsertCache(TModel model)`, `Future<void> markSynced(String id)`, `DateTime? syncedAtOf(TModel model)` (extracts the `synced_at` field so the base class can check staleness without knowing the model's shape), `Future<void> writeRemote(TModel model)`, `Future<void> writeCache(TModel model)`.
- Concrete method `Future<TModel?> get(String id) async { ... }` implementing exactly the cache-first/stale-fallback flow from 01-RESEARCH.md's `getSet()` example: try `getCached`, if present and not stale (`sync_metadata.dart`'s `isStale()`) return it; else `getRemote`, on success `upsertCache` and return; on `FirebaseException`, fall back to `getCached` (even if stale) and return that, or `rethrow` if no cache exists at all.
- Concrete method `Future<void> save(TModel model) async { ... }` implementing Firestore-first write: `writeRemote(model)`, then `writeCache(model)`, then `markSynced(<id extracted via a required idOf(TModel) hook>)`.

`sync_metadata.dart`: a standalone `bool isStale(DateTime? syncedAt, {Duration maxAge = const Duration(hours: 1)})` function (default 1-hour staleness window per 01-RESEARCH.md's `_isStale()` example) — kept separate from the abstract class so it's independently unit-testable and reusable by code that isn't a `BaseSyncRepository` subclass.

`lib/core/utils/failure.dart`: a small sealed/abstract `Failure` class hierarchy (`NetworkFailure`, `CacheMissFailure`, `UnknownFailure`) for repositories to translate `FirebaseException`/`DatabaseException` into app-domain errors — per ARCHITECTURE.md § E "Base Classes & Extensions" (`lib/core/utils/failure.dart — Failure sealed class`). Keep minimal: 3 variants, each holding a `String message`.
