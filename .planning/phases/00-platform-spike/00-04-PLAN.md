## Phase Goal

**As a** developer about to build 8 phases of production code on this stack, **I want to** see proof that Firebase (Auth + Firestore) and `sqflite_common_ffi` actually initialize and perform real read/write work on both an Android emulator and Windows desktop, **so that** the team does not commit weeks of work to a stack assumption that silently fails on one platform.

<objective>
Get an Android emulator running (human-required per ground truth — none exists on this machine), then run the identical spike app against it and capture the same evidence captured for Windows in Plan 03. This is where the `10.0.2.2` vs `localhost` host-address gotcha either proves out or fails — the single most likely reason criterion 4 could fail on Android for a non-real reason, per the user's locked decision.

Purpose: Prove or disprove FND-01/FND-03/FND-04 on Android with captured command output. Sequenced after Plan 03 (not parallel) so only one Firebase Emulator Suite instance runs at a time — avoids port collisions between concurrent Windows and Android verification runs.
Output: spike_platform/android_run_log.txt (captured evidence) and a human-confirmed visual check.
</objective>
