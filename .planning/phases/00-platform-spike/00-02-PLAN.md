## Phase Goal

**As a** developer about to build 8 phases of production code on this stack, **I want to** see proof that Firebase (Auth + Firestore) and `sqflite_common_ffi` actually initialize and perform real read/write work on both an Android emulator and Windows desktop, **so that** the team does not commit weeks of work to a stack assumption that silently fails on one platform.

<objective>
Implement the two real test paths (SQLite round-trip, Firebase Auth+Firestore round-trip) against the contracts Plan 01 created, and wire them into a single-screen UI that auto-runs both tests on launch and exposes re-run buttons plus a visible log — the thinnest real (not stubbed) end-to-end slice per MVP vertical-slice rules. Every test function catches its own exceptions and returns a PASS/FAIL string instead of throwing, so a genuine platform failure shows up as evidence in the log rather than crashing the whole spike app (which would make it impossible to distinguish "the app crashed" from "the feature failed").

Purpose: This is the only plan that writes the actual behavior being tested. Wave 3/4 only run it and capture evidence — no new spike logic is written after this plan.
Output: spike_platform/lib/sqlite_service.dart, spike_platform/lib/firebase_service.dart, spike_platform/lib/main.dart — a runnable (not yet run) spike app.
</objective>
