## Phase Goal

**As a** developer about to build 8 phases of production code on this stack, **I want to** see proof that Firebase (Auth + Firestore) and `sqflite_common_ffi` actually initialize and perform real read/write work on both an Android emulator and Windows desktop, **so that** the team does not commit weeks of work to a stack assumption that silently fails on one platform.

<objective>
Actually run the spike app on Windows desktop against a live Firebase Emulator Suite and capture the runtime evidence — this is the first of two platform verification waves (Windows is verified first because `flutter devices` already lists it; no Android emulator is configured yet per ground truth, which Wave 4 handles).

Purpose: Prove or disprove FND-02/FND-03/FND-04 on Windows with captured command output, not source-code inspection. A FAIL result here is a legitimate, valuable spike finding — it is NOT a failure of this plan's task, only an uncaptured/uncrashed run would be.
Output: spike_platform/windows_run_log.txt (captured evidence) and a human-confirmed visual check.
</objective>

  </action>
  <acceptance_criteria>
