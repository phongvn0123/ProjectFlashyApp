  <action>
    Tự động hoá tối đa bằng Bash (per Automation-first rule — không giao việc CLI cho người dùng nếu Claude tự chạy được):
    1. Xác nhận có thiết bị Android khả dụng: `adb devices`. Nếu chưa có emulator nào chạy, liệt kê AVD bằng `emulator -list-avds`, khởi động một AVD bằng `emulator -avd <name> &` (chạy nền), rồi `adb wait-for-device` và poll `adb shell getprop sys.boot_completed` tới khi in ra `1`.
    2. Xác định loại system image của AVD đang chạy: lấy tên AVD qua `adb -s <serial> emu avd name`, sau đó đọc `tag.id` trong file `<thư mục AVD>/<name>.avd/config.ini` (trên Windows dưới Git Bash: `$USERPROFILE/.android/avd/<name>.avd/config.ini`). Nếu `tag.id` KHÔNG chứa `google_apis`, ghi nhận đây là blocker môi trường (Firebase Auth sẽ fail) để báo cáo NO-GO ở Task 3 — không tự ý cài lại system image mới vì tốn thời gian/băng thông ngoài phạm vi tự động hoá của spike.
    3. Khởi động Firebase Emulator Suite ở chế độ nền (giữ quyết định "manual start" của researcher chỉ áp dụng cho người vận hành thủ công; ở đây Claude tự động hoá thay): `cd spike_platform && firebase emulators:start --only auth,firestore --project demo-spike-project > firebase_emulator.log 2>&1 &`. Poll `firebase_emulator.log` tới khi xuất hiện dòng `All emulators ready!` (timeout ~60 giây; nếu quá timeout, ghi nhận NO-GO).
    4. Chạy app: `cd spike_platform && flutter run -d android --no-pub > spike_run.log 2>&1 &` (chạy nền). Poll `spike_run.log` tới khi xuất hiện CẢ HAI dòng bắt đầu bằng `[SPIKE] SQLITE` và `[SPIKE] FIREBASE` (PASS hoặc FAIL) — timeout ~120 giây.
    5. Tạo thư mục `.planning/phases/00-platform-spike/evidence/` nếu chưa có. Chụp màn hình emulator bằng `adb exec-out screencap -p > .planning/phases/00-platform-spike/evidence/spike-screenshot.png`.
    6. Đọc lại file ảnh vừa chụp (dùng Read tool — hỗ trợ ảnh) để tự xác nhận cả 2 dòng PASS/FAIL hiển thị trên UI khớp với `spike_run.log`.
    7. Dừng tiến trình: kill tiến trình nền `flutter run` (theo PID đã lưu khi khởi động ở bước 4), kill tiến trình nền `firebase emulators:start` (theo PID đã lưu ở bước 3). Xác nhận không còn tiến trình lắng nghe port 9099/8080 bằng `netstat -ano | grep -E "9099|8080"` (kỳ vọng output rỗng) — mitigation cho threat T-0-05.
  </action>
