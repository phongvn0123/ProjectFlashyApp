2.1 
1.1.1 Login System

Primary Actors
 | Admin
 | 
 | 
 | 
Description
 | As a guest, I want to register an account so that I can access system features. 
 | 
Preconditions
 | Guest has access to the registration page;
 Email/Username does not already exist in the system. 
 | 
Postconditions
 | New account is created successfully; User information is stored in the system database. 
 | 
Normal Sequence/Flow
 | Registration System 
1. Guest accesses the Register page.
2. System displays the registration form.
3. Guest enters registration information: Username, Email, Password, Confirm Password.
4. Guest clicks the Register button.
5. System validates the entered information.
6. System creates a new account.
7. System displays a success message.
8. System redirects Guest to the Login page. 
 | 
Alternative Sequences/Flows
 | Step 5.1_Invalid Registration Information 
1. System displays an appropriate error message in one of the following cases: Email already exists; Passwords do not match; Required fields are empty.
2. Guest corrects the information.
3. Return to Step 3 of the Normal Flow. 
Step 6.1_System Error 
1. System fails to create the account.
2. System displays an error notification.
3. Use case ends. 
 |

2.1.16 Create Flashcard Set

Primary Actors
 | Student, Teacher 
 | Secondary Actors
 | None 
 | 
Description
 | As a user, I want to be able to create a flashcard set so that I can organize learning materials and study effectively. 
 | 
Preconditions
 | User has logged into the system.
User has permission to create flashcard sets.
 | 
Postconditions
 | A new flashcard set is created successfully.
The flashcard set is stored in the system.
 | 
Normal Sequence/Flow
 | Create Flashcard Set 
User clicks the Create Set button.
System displays the Create Flashcard Set screen.
User enters flashcard set information (title, description, visibility, etc.).
User adds one or more flashcards to the set.
User clicks the Save/Create button.
System validates the entered information 
System creates the flashcard set.
System redirects the user to the Flashcard Set Detail page.

 | 
Alternative Sequences/Flows
 | Step 4.1_Add New Flashcard 
User clicks the Add Flashcard button.
System displays a new flashcard form.
User enters flashcard information.
System adds the flashcard to the current set draft.
Return to step 4 of normal flow.
Step 6_System cannot create the flashcard set 
The title field is blank 
No flashcard is added to the set 
The flashcard set title exceeds the allowed length 
System error occurs while saving data
